# Работа со счетами Mail.ru

## Правила

- Работать через открытую вкладку Mail.ru в Microsoft Edge и её авторизованный сеанс.
- Для Edge брать управление Chromium-браузером через навык Chrome. Не подменять Edge Яндекс Браузером, Chrome или встроенным браузером.
- Не просить пользователя скачать страницы или письма, пока вкладка доступна.
- Не менять письма, флаги, папки и поиск без прямой просьбы.
- Не выводить почту, ФИО, телефон, адрес и реквизиты.

## Подключение к нужной вкладке

1. Подключиться по точному URL Mail.ru через `getForUrl`.
2. Проверить, что выбранное подключение видит вкладку `e.mail.ru`. Название браузера в служебном ответе может быть общим: решать по URL открытых вкладок.
3. Взять точную вкладку из `openTabs()` и вызвать `claimTab`. Если она уже под управлением, взять её из `tabs.list()` через `tabs.get()`.
4. Не угадывать ID вкладки и не брать первую вкладку в списке.
5. Оставить исходный поиск открытым. Для обхода писем создать рабочую вкладку с тем же URL поиска.

Если вкладка открыта в приватном окне без доступа расширения, попросить открыть тот же поиск в обычном окне Edge. Не переходить в другой браузер без просьбы.

## Обход писем

1. Снять DOM-снимок поисковой выдачи.
2. Найти письмо по дате, отправителю, теме или имени вложения.
3. Открывать письмо кликом по точной ссылке из выдачи. Не строить прямой URL письма, если ссылка уже есть в DOM.
4. После открытия дать интерфейсу Mail.ru коротко догрузиться и снять новый DOM-снимок.
5. Возвращаться к выдаче через `back()` и снова брать свежий DOM перед следующим письмом.

Не ходить по письмам через пользовательскую исходную вкладку: рабочая вкладка защищает открытый поиск от сбоя или смены страницы.

## Скачивание PDF

1. На странице письма найти точную ссылку вложения с именем PDF и `mode=attachment`.
2. Проверить, что локатор один.
3. Запустить ожидание события `download`.
4. Кликнуть ссылку с модификатором `Alt`.
5. Убедиться, что файл появился в `%USERPROFILE%\Downloads` и имеет ненулевой размер.

Не применять `downloadMedia()` к вложениям Mail.ru в Edge: оно может открыть `af.attachmail.ru`, который Edge блокирует с `ERR_BLOCKED_BY_CLIENT`. Не открывать прямой URL PDF ради скачивания.

Если нужно скачать несколько счетов, повторить этот цикл для каждого письма. Не считать письмо обработанным, пока файл не найден на диске.

## Сбор данных

- Брать дату из самого счета, а не из времени письма.
- Брать количество, цену, сумму, ставку и сумму НДС из строки товара и итогов счета.
- Для сканов отрисовать PDF в PNG и проверить страницу глазами.
- Если пользователь задал колонки Markdown, не добавлять лишние колонки.
- Сохранять итоговый Markdown туда, куда просит пользователь; по умолчанию — на рабочий стол.

## Проверка БД — только по просьбе

Не читать БД и не искать отсутствующие файлы при обычном сборе писем и счетов.

Если пользователь прямо просит сверить с Nocombro:

- открыть заново `%APPDATA%\Nocombro\nocombro.db` в режиме чтения перед каждым итоговым выводом;
- сверить дату, количество, цену за кг, НДС, доставку, себестоимость и `file.owner_type='TRANSACTION'`;
- сообщать об отсутствующем PDF только при такой проверке;
- учитывать, что пользователь мог исправить запись во время работы: не опираться на старый результат запроса;
- по просьбе показать только ошибки и не перечислять верные строки.

Не обращаться к YDB без прямой просьбы.

## Завершение

- Оставить исходную вкладку поиска открытой как `handoff`.
- Закрыть рабочие, PDF и заблокированные вкладки через `tabs.finalize()`.
- В ответе сообщить результат, путь к файлам и пропуски. Не описывать служебные шаги подключения.

Примеры вызова:

- `$audit-mailru-invoices собери счета из открытого поиска Mail.ru и скачай PDF`;
- `$audit-mailru-invoices открой письма этого отправителя и сделай таблицу дата, количество, НДС`;
- `$audit-mailru-invoices сверяй скачанные счета с локальной БД и покажи только ошибки`.
