# Проверка заказов Ингремарт

## Правила

- Работать в режиме чтения. Не менять БД, сайт и файлы без прямой просьбы.
- Использовать открытую авторизованную вкладку Ингремарт в Яндекс Браузере. Для управления брать навык Chrome: расширение видит Яндекс как Chromium-браузер.
- Не обращаться к YDB. Локальная БД: `%APPDATA%\Nocombro\nocombro.db`.
- Не выводить ФИО, почту, телефон, адрес, реквизиты и иные личные данные.
- При сравнении даты не учитывать время: пользователю важен только календарный день.
- Допущение пользователя: декларация ООО «РУСТАРК» у мальтодекстрина допустима. Не считать это ошибкой без новых данных.

## Сбор данных с сайта

1. Открыть список заказов и взять номера со скриншота или из запроса.
2. Заходить в заказ по дате либо по адресу:
   `https://ingremart.biz/index.php?dispatch=orders.details&order_id=НОМЕР`.
3. Для каждого заказа записать:
   - статус и дату;
   - название, артикул, фасовку, цену за кг и массу каждой строки;
   - сумму товаров;
   - строку `Скидка на заказ`;
   - ставку и сумму НДС;
   - доставку и итог к оплате.
4. Не вычитать бонусы или баллы отдельно. Учитывать только явную строку `Скидка на заказ`.
5. Черновик и отклонённый заказ не должны давать покупку в БД. Заказ `В пути` можно занести, но завершённую локальную покупку отметить как риск, пока товар не принят.
6. После работы оставить исходную вкладку списка заказов открытой, а созданные вкладки закрыть.

## Сопоставление с локальной БД

Сначала получить свежие покупки:

```powershell
$db = Join-Path $env:APPDATA 'Nocombro\nocombro.db'
sqlite3 -readonly $db ".headers on" ".mode tabs" `
  "SELECT id, created_at, is_completed, updated_at FROM transact WHERE transaction_type='BUY' AND deleted_at IS NULL ORDER BY id DESC LIMIT 30;"
```

Сопоставлять заказ с `transact.id` по дате, составу, массе и общей сумме. Не полагаться лишь на порядок строк.

Основные связи:

- `transact.id -> buy.transaction_id`;
- `buy.movement_id -> batch_movement.id`;
- `batch_movement.batch_id -> batch.id`;
- `batch.product_id -> product.id`;
- `batch.id -> batch_cost_price.batch_id`;
- `expense.transaction_id -> transact.id`;
- `file.owner_type='TRANSACTION'` и `file.owner_id=transact.id`.

Единицы хранения:

- `buy.price` и `batch_cost_price.cost_price_per_unit` — копейки за кг;
- `expense.amount` — копейки;
- `batch_movement.count / 1000` — кг;
- у покупки движение имеет тип `INCOMING`.

## Проверка суммы и себестоимости

В принятой пользователем схеме скидку вычитать из доставки:

```text
расход для покупки = доставка − скидка на заказ
```

Проверка итога:

```text
сумма товаров + расход для покупки = итог к оплате
```

Приложение делит общий расход поровну на каждый килограмм заказа:

```text
себестоимость строки, ₽/кг = цена строки, ₽/кг + расход для покупки, ₽ / общий вес заказа, кг
```

В БД формула в целых единицах:

```text
expected_cost_kop = round(buy.price + expense.amount * 1000 / total_count)
```

Сверить сохранённый `batch_cost_price.cost_price_per_unit` с этой формулой и с суммой сайта.

## Что проверять

- дата покупки совпадает с датой заказа;
- состав, артикулы, масса и цены совпадают построчно;
- расход имеет тип `TRANSPORT_DELIVERY`;
- расход равен доставке минус скидка;
- НДС каждой строки совпадает со счётом: исторические 20% и 22% не смешивать;
- дата партии не позже даты покупки;
- сохранённая себестоимость совпадает с расчётом;
- прикреплённый счёт PDF есть на диске и относится к нужному заказу;
- локальная завершённость не противоречит статусу сайта.

Если пользователь просит полную проверку, также выполнить `PRAGMA quick_check`, `PRAGMA foreign_key_check` и поиск дублей `sync_id`. При просьбе проверить только правки не расширять проверку.

## Ответ пользователю

Писать кратко и без личных данных. Для ошибок использовать таблицу:

| Заказ | ID записи | Что исправить |
|---|---:|---|
| #НОМЕР | ID | Поле: текущее значение → верное значение |

Сначала перечислить ошибки. Верные записи можно собрать одной строкой. Всегда сказать, менялись ли данные.

Примеры вызова:

- `$audit-ingremart-orders проверь заказы на скриншоте, только локально`;
- `$audit-ingremart-orders перепроверь только внесённые исправления`.
