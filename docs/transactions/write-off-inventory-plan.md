# План поддержки списаний и инвентаризаций

## Границы этапа

На текущем этапе новые операции `WRITE_OFF` и `INVENTORY` не реализуются. До полной поддержки их нельзя выбрать при создании транзакции. Уже сохранённые строки должны открываться без падения в режиме «тип пока не поддержан».

## WRITE_OFF

- Списание использует существующее движение `BatchMovement.OUTGOING`.
- Комментарий хранится в общем поле `Transact.comment`.
- Новый запрет отрицательного остатка не вводится: действуют текущие правила движения партий.

## INVENTORY

Для строк инвентаризации нужна новая сущность `InventoryLine` со связями и полями:

- транзакция;
- партия;
- движение;
- ожидаемый остаток;
- фактический остаток;
- sync-метаданные.

Пользователь выбирает партии и вводит фактический остаток. Строки с нулевым фактическим остатком хранятся.

Правка старой инвентаризации пересчитывает её и все более поздние инвентаризации по сохранённым фактическим остаткам.

## Будущий этап

Полная поддержка включает:

- миграцию Room `8 → 9`;
- mirror-таблицу в YDB;
- вкладки формы;
- DI;
- tombstones;
- тесты создания, правки, удаления, sync и пересчёта истории.

Эти работы не входят в текущий набор правок.
